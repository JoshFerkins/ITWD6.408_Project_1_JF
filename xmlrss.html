XML Syntax Rules

XML has specific rules that must be followed, XML may be easy to write but these must be followed to provide, functionality, readbility, and accessibility for users and developers.

Closing Tags

When ever a tag is created it must also be created with a closing tag. Websites ran without closing tags will not know when the specific content is supposed to finish

<title>This is an article title<title>

ROOT Elements

Elements that contain content are required to be nested within a root element. Root elements contain parent elements which also can contain other child elements.

<root><element>This is a nested element</element></root>

Name / Value pairs

When creating attributes for elements you must use a name / value pair. This servers as a key tag for nested child elements.

<element class="main">This is a name/value pair</element>

Quoted Values

Values created through name / value pairs must be created within double quotes " " following the name and '=' sign.

<element class="main">This is a quoted value</element>

Preserved White Spaces

When creating content within elements, it is important to be aware that white spaces are preserved in the text. any white space you enter will not be removed / ignored

<text>This is displaying preserved white space</text>

Special Characters

When writing special characters, "& * %...", it is important to know that they are written in a special format to be displayed as text. Typically special characters perform actions within text unless written as so

<element>"&" must be written as & to display</element>
